# 🌊 Seahorse-Village (해마촌)
> **웹 개발의 기초를 다지기 위해 처음으로 구현한 정적 웹사이트**

HTML과 CSS의 기본 구조를 이해하고, 디자인을 코드로 구현하는 과정을 경험하기 위해 제작한 첫 번째 프론트엔드 프로젝트입니다.

## 📌 주요 학습 포인트 (Learning Points)
*   **HTML 구조 설계**: 웹 페이지의 뼈대를 구성하는 시맨틱 태그의 기초 활용
*   **CSS 스타일링**: 색상, 폰트, 레이아웃 배치를 통한 기본적인 웹 디자인 구현
*   **하이퍼링크 연결**: 여러 HTML 페이지(`home`, `contact`) 간의 유기적인 이동 구현
*   **자산 관리**: 이미지 리소스를 별도 폴더로 관리하여 프로젝트 구조화 연습

## 🛠 사용 기술 (Tech Stack)
*   **Language**: HTML5, CSS3
*   **Tools**: Visual Studio Code

## 📂 프로젝트 구조 (Project Structure)
```text
├── image/           # 프로젝트 사용 이미지 리소스
├── home.html        # 메인 페이지
├── contact.html     # 문의 및 소개 페이지
└── style.css        # 전체 페이지 스타일 시트

## 🧐 아쉬운 점 및 회고 (Retrospective)

### 1. 코드의 중복과 유지보수
처음 만든 프로젝트라 CSS 파일 하나(`style.css`)에 모든 스타일을 다 넣다 보니, 나중에는 어떤 코드가 어떤 화면에 영향을 주는지 파악하기 어려웠습니다. 
*   **교훈**: 이후 프로젝트에서는 **CSS 변수**를 활용하고, 스타일을 컴포넌트 단위로 분리하는 법을 익혔습니다.

### 2. 반응형 디자인의 미숙함
PC 화면 위주로 제작하다 보니 모바일에서 레이아웃이 깨지는 현상이 있었습니다. 
*   **교훈**: 이를 계기로 **미디어 쿼리(Media Query)**와 **Flexbox**를 집중적으로 공부하여 다음 프로젝트(`Dev-Showcase-Portfolio`)에서는 모든 기기에 대응하는 반응형 웹을 구현했습니다.

### 3. 디렉토리 구조의 단순함
파일이 적을 때는 괜찮았지만, 규모가 커질수록 관리하기 힘든 구조임을 깨달았습니다. 
*   **교훈**: 폴더 구조를 `assets`, `pages`, `styles` 등으로 나누어 관리하는 **프로젝트 구조화**의 중요성을 배웠습니다.
